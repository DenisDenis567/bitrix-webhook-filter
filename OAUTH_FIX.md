# 🔧 Исправление ошибки OAuth в Google Apps Script

## Проблема

```
You are receiving this error either because your input OAuth2 scope name is invalid
or it refers to a newer scope that is outside the domain of this legacy API.
```

## Причина

В файле `appsscript.json` указан **устаревший или неправильный** OAuth scope для `UrlFetchApp.fetch()`.

---

## ✅ Решение

### Шаг 1: Откройте Google Apps Script

1. Откройте вашу Google Таблицу "Реестр платежей"
2. Нажмите **Расширения** → **Apps Script**

### Шаг 2: Найдите файл `appsscript.json`

1. В левой панели найдите файл **`appsscript.json`**
2. Если его нет:
   - Нажмите на шестерёнку ⚙️ (Настройки проекта)
   - Включите **"Показывать файл манифеста appsscript.json в редакторе"**
   - Вернитесь к редактору файлов

### Шаг 3: Замените содержимое `appsscript.json`

**Удалите всё** и вставьте этот код:

```json
{
  "timeZone": "Asia/Vladivostok",
  "dependencies": {},
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8"
}
```

**ВАЖНО:** Мы **удалили секцию `oauthScopes`** полностью. Google Apps Script автоматически определит нужные разрешения на основе используемых API (`UrlFetchApp`, `SpreadsheetApp`).

### Шаг 4: Сохраните изменения

Нажмите **Ctrl+S** или кликните на иконку дискеты.

### Шаг 5: Удалите старые триггеры

1. Нажмите на иконку **⏰ (Триггеры)** в левой панели
2. Удалите все триггеры для функции `onEdit` (если есть)
3. Закройте страницу триггеров

### Шаг 6: Повторно авторизуйте скрипт

1. Вернитесь в вашу Google Таблицу
2. Найдите любую строку с пустой ячейкой в **колонке H** (Оплачен)
3. **Введите слово `Оплачен`** и нажмите Enter

**Должно появиться окно авторизации!**

4. Нажмите **"Продолжить"** → выберите ваш Google аккаунт
5. Нажмите **"Дополнительно"** (если будет предупреждение)
6. Нажмите **"Перейти к ... (небезопасно)"**
7. Нажмите **"Разрешить"**

### Шаг 7: Проверьте работу

После успешной авторизации должно появиться уведомление:

```
✅ Успешно отправлено
CRM ID XXX добавлен в очередь.

Элемент будет переведён на стадию "Оплачено" в Bitrix24
в ближайший часовой запуск.

Элементов в очереди: 1
```

## 🧪 Альтернативный тест (если основной не сработал)

1. В редакторе Apps Script найдите функцию **`testWebhook()`**
2. Выберите её в выпадающем списке сверху
3. Нажмите **▶ Выполнить**
4. Дайте разрешения, если попросит
5. Проверьте логи (Ctrl+Enter) - должно быть `✓ Тест пройден успешно!`

---

## 📋 Что дальше?

После успешной авторизации:

- Каждый раз при вводе `Оплачен` в колонке H скрипт будет автоматически:
  - Брать ID CRM из колонки M
  - Отправлять его на сервер Railway
  - Добавлять в очередь

- **Каждый час в :00 минут** сервер автоматически:
  - Обработает все элементы из очереди
  - Переведёт их на стадию `DT1064_28:SUCCESS` в Bitrix24

---

## ❓ Если всё равно не работает

Проверьте логи Apps Script:

1. В редакторе Apps Script нажмите **Ctrl+Enter** (или кнопку "Журнал выполнения")
2. Посмотрите, есть ли ошибки
3. Если ошибка про `UrlFetchApp` - повторите шаги 3-7 заново
