# Настройка Google Sheets для интеграции с Bitrix24

Этот документ описывает, как настроить Google Таблицу "Реестр платежей" для автоматической отправки webhook при изменении статуса оплаты.

---

## 📋 Структура таблицы

Таблица должна содержать следующие колонки:

| Колонка | Название | Описание |
|---------|----------|----------|
| A | дата | Дата создания записи |
| B | Получатель | Кому платим |
| C | За что платим | Описание платежа |
| D | Код | Код платежа |
| E | Сумма | Сумма платежа |
| F | Важность | Приоритет |
| G | Согласован | Статус согласования |
| **H** | **Оплачен** | **ТРИГГЕР: статус оплаты** |
| I | дата оплаты | Дата фактической оплаты |
| J | Инициатор счета | Кто инициировал |
| K | крайний срок оплаты счета | Дедлайн |
| L | Комментарии | Примечания |
| **M** | **ID CRM** | **ID элемента из Bitrix24 (обязательно!)** |

## 🔧 Установка Google Apps Script

### Шаг 1: Открыть редактор скриптов

1. Откройте вашу Google Таблицу "Реестр платежей"
2. Нажмите **Расширения** → **Apps Script**
3. Откроется редактор кода

### Шаг 2: Вставить код

Удалите весь существующий код и вставьте следующий:

```javascript
/**
 * Триггер, который срабатывает при редактировании ячейки
 */
function onEdit(e) {
  const sheet = e.source.getActiveSheet();
  const range = e.range;
  const col = range.getColumn();

  // Колонка H = 8 (Оплачен)
  if (col === 8 && e.value === "Оплачен") {
    const row = range.getRow();
    const crmId = sheet.getRange(row, 13).getValue(); // Колонка M (ID CRM)

    // Пропускаем строки без ID CRM
    if (!crmId) {
      Logger.log(`Строка ${row}: нет ID CRM, пропускаем`);
      return;
    }

    // Формируем данные для отправки
    const payload = {
      crmId: String(crmId),
      status: "paid",
      timestamp: new Date().toISOString(),
      sheetRow: row
    };

    // URL вашего сервера на Railway
    const url = "https://bitrix24-webhook-filter-production.up.railway.app/sheets-webhook";

    try {
      const response = UrlFetchApp.fetch(url, {
        method: "POST",
        contentType: "application/json",
        payload: JSON.stringify(payload),
        muteHttpExceptions: true
      });

      const statusCode = response.getResponseCode();

      if (statusCode === 200) {
        Logger.log(`✓ Успешно отправлено: CRM ID ${crmId}, строка ${row}`);
      } else {
        Logger.log(`✗ Ошибка отправки (код ${statusCode}): ${response.getContentText()}`);
      }
    } catch (err) {
      Logger.log(`✗ Критическая ошибка отправки: ${err.message}`);
    }
  }
}
```

### Шаг 3: Сохранить и назвать проект

1. Нажмите **💾 (Сохранить)**
2. Назовите проект, например: `Bitrix24 Payment Sync`

### Шаг 4: Дать разрешения

При первом запуске Google попросит разрешения:

1. Нажмите **Просмотреть разрешения**
2. Выберите свой Google аккаунт
3. Нажмите **Дополнительно** → **Перейти к "Bitrix24 Payment Sync" (небезопасно)**
4. Нажмите **Разрешить**

## ⚠️ Важные моменты

### 1. Точное совпадение слова "Оплачен"
Триггер срабатывает **только** если в колонке H написано именно **"Оплачен"** (с большой буквы).

Если нужны другие варианты (например, checkbox или дата), измените строку в коде:
```javascript
if (col === 8 && e.value === "Оплачен") {
```

### 2. Обязательное поле ID CRM
Если в колонке **M** нет ID элемента из Bitrix24, webhook **не отправится**.

### 3. Работа с несколькими вкладками
Скрипт работает **на всех вкладках** таблицы. Если нужно ограничить только одной вкладкой (например, "Март 2025"), добавьте проверку:

```javascript
function onEdit(e) {
  const sheet = e.source.getActiveSheet();
  const sheetName = sheet.getName();

  // Ограничиваем только вкладкой "Март 2025"
  if (sheetName !== "Март 2025") {
    return;
  }

  // ... остальной код ...
}
```

### 4. Частота обработки
Элементы из очереди отправляются в Bitrix24 **каждый час в :00 минут**.

Если нужна другая частота:
- `*/30 * * * *` — каждые 30 минут
- `0 */2 * * *` — каждые 2 часа
- `0 9,17 * * *` — в 9:00 и 17:00

Измените строку в [src/server.js](src/server.js:283):
```javascript
cron.schedule('0 * * * *', async () => {
```

## 🐛 Устранение проблем

### Webhook не отправляется

1. **Проверьте колонку H**: написано ли точно "Оплачен"?
2. **Проверьте колонку M**: заполнен ли ID CRM?
3. **Откройте логи Apps Script**: Расширения → Apps Script → Выполнения
4. **Проверьте URL сервера**: правильно ли указан домен Railway?

### Сервер возвращает ошибку 400

Проверьте формат отправляемых данных. Должно быть:
```json
{
  "crmId": "816",
  "status": "paid"
}
```

### Элемент не переводится на стадию в Bitrix24

1. Проверьте переменные окружения на Railway:
   - `BITRIX_WEBHOOK_URL` — правильный ли webhook?
   - `BITRIX_PAID_STAGE_ID` — правильный ли ID стадии?
2. Проверьте логи на Railway: `railway logs`
3. Попробуйте вручную вызвать Bitrix24 API через Postman/curl
