## 🎯 Что нужно для начала

### Требования:

- ✅ Google Таблица "Реестр платежей" с нужной структурой
- ✅ Сервер на Railway уже задеплоен и работает
- ✅ URL сервера: `https://bitrix24-webhook-filter-production.up.railway.app`

### Структура Google Таблицы:

| Колонка | Название | Описание |
|---------|----------|----------|
| E | Сумма | Сумма платежа |
| H | Оплачен | Статус оплаты (триггер) |
| I | дата оплаты | Дата оплаты |
| M | ID CRM | ID элемента в Bitrix24 CRM |

**Важно:** Колонка M должна содержать числовой ID элемента из Bitrix24!

## 📝 Установка скрипта

### Шаг 1: Откройте редактор Apps Script

1. Откройте вашу Google Таблицу **"Реестр платежей"**
2. В меню выберите: **Расширения** → **Apps Script**
3. Откроется редактор Apps Script в новой вкладке

### Шаг 2: Скопируйте код скрипта

1. Откройте файл [google-apps-script.js](google-apps-script.js) в этом репозитории
2. Нажмите **Ctrl+A** (выделить всё) → **Ctrl+C** (скопировать)

### Шаг 3: Вставьте код в редактор

1. В редакторе Apps Script выберите файл **Code.gs** (или Код.gs)
2. **Удалите** весь существующий код (обычно там `function myFunction() { }`)
3. Нажмите **Ctrl+V** (вставить скопированный код)
4. Нажмите **Ctrl+S** или кликните на иконку дискеты (Сохранить)

### Шаг 4: Проверьте конфигурацию

Найдите в коде секцию `CONFIG` (примерно строка 22):

```javascript
const CONFIG = {
  // URL вашего сервера на Railway
  SERVER_URL: "https://bitrix24-webhook-filter-production.up.railway.app/sheets-webhook",

  // Номера колонок (не менять без необходимости!)
  COLUMNS: {
    STATUS: 8,      // Колонка H - "Оплачен"
    CRM_ID: 13,     // Колонка M - "ID CRM"
    DATE: 9,        // Колонка I - "дата оплаты"
    AMOUNT: 5       // Колонка E - "Сумма"
  },

  // Значение, которое триггерит отправку
  TRIGGER_VALUE: "Оплачен"
};
```

**Проверьте:**
- `SERVER_URL` - должен быть URL вашего сервера на Railway
- `COLUMNS.STATUS` - номер колонки "Оплачен" (H = 8)
- `COLUMNS.CRM_ID` - номер колонки "ID CRM" (M = 13)
- `TRIGGER_VALUE` - слово, которое триггерит отправку ("Оплачен")

**Если структура вашей таблицы другая** - измените номера колонок!

## 🔐 Настройка OAuth разрешений

Google Apps Script требует правильных OAuth разрешений для отправки HTTP запросов.

### Шаг 1: Проверьте файл манифеста

1. В редакторе Apps Script нажмите на **⚙️ (Настройки проекта)** слева
2. Убедитесь, что галочка **"Показывать файл манифеста appsscript.json в редакторе"** включена
3. Вернитесь к файлам (иконка **< >** слева)
4. Должен появиться файл **appsscript.json**

### Шаг 2: Настройте appsscript.json

1. Откройте файл **appsscript.json**
2. **Убедитесь, что там НЕТ секции `oauthScopes`!**

Правильный `appsscript.json` должен выглядеть так:

```json
{
  "timeZone": "Asia/Vladivostok",
  "dependencies": {},
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8"
}
```

**Если есть секция `oauthScopes` - удалите её!**

Google Apps Script автоматически определит нужные разрешения на основе используемых API.

3. Сохраните файл (Ctrl+S)

## ⚙️ Создание триггера

**ВАЖНО:** Простой триггер `onEdit()` не имеет достаточных разрешений для HTTP запросов. Нужно создать **установленный триггер**.

### Шаг 1: Откройте страницу триггеров

1. В редакторе Apps Script нажмите на **⏰ (Триггеры)** в левой панели
2. Если есть старые триггеры для `onEdit` - **удалите их** (три точки → Удалить триггер)

### Шаг 2: Создайте новый триггер

1. Нажмите кнопку **+ Добавить триггер** (внизу справа)
2. В появившемся окне настройте параметры:

**Обязательные настройки:**

| Параметр | Значение |
|----------|----------|
| **Выберите функцию** | `onEdit` |
| **Выберите, какое развертывание выполнить** | `Основное развертывание` |
| **Выберите источник события** | `Из таблицы` |
| **Выберите тип события** | `При редактировании` |

**Опциональные настройки:**

| Параметр | Значение |
|----------|----------|
| **Настройки уведомлений об ошибках** | `Каждый день` (или по желанию) |

3. Нажмите кнопку **Сохранить**

### Шаг 3: Дайте разрешения

**Должно появиться окно авторизации!**

1. Нажмите **"Продолжить"**
2. Выберите ваш Google аккаунт
3. Возможно появится предупреждение **"Это приложение не проверено"**:
   - Нажмите **"Дополнительно"** (внизу слева)
   - Нажмите **"Перейти к [название проекта] (небезопасно)"**
4. Прочитайте список разрешений
5. Нажмите **"Разрешить"**

**Разрешения, которые запросит скрипт:**
- Просмотр и управление таблицами, в которых установлено это приложение
- Подключение к внешнему сервису (для отправки HTTP запросов)

### Шаг 4: Проверьте триггер

После успешной авторизации вы вернётесь на страницу триггеров.

Должен появиться триггер с параметрами:

| Параметр | Значение |
|----------|----------|
| **Функция** | onEdit |
| **Событие** | Из таблицы - При редактировании |
| **Развертывание** | Основное развертывание |
| **Владелец** | Ваш email |

✅ **Триггер создан и авторизован!**

## 🧪 Тестирование

### Тест 1: Тестовая функция (рекомендуется сначала)

1. В редакторе Apps Script выберите функцию **`testWebhook`** в выпадающем списке сверху
2. Нажмите **▶ Выполнить**
3. Дождитесь выполнения (5-10 секунд)
4. Проверьте логи: **Ctrl+Enter** или **View → Журнал выполнения**

**Ожидаемый результат в логах:**

```
Отправка тестового webhook...
HTTP 200: {"success":true,"queueSize":1,"message":"Added to queue..."}
✓ Успех: CRM ID 999 добавлен в очередь. Всего в очереди: 1
✓ Тест пройден успешно!
```

**Если тест успешен** → переходите к Тесту 2!

**Если ошибка** → см. раздел [Устранение неполадок](#устранение-неполадок)

### Тест 2: Реальное редактирование таблицы

1. Откройте вашу Google Таблицу "Реестр платежей"
2. Найдите строку, где:
   - В колонке **M (ID CRM)** есть числовой ID элемента из Bitrix24
   - В колонке **H (Оплачен)** пусто или другое значение (не "Оплачен")
3. Введите в ячейку колонки H слово: **`Оплачен`**
4. Нажмите Enter

**Ожидаемое поведение:**

✅ **НЕТ всплывающих окон!** - скрипт работает тихо в фоне (как Make.com)

**Проверка успешной отправки:**

1. Откройте **Расширения → Apps Script**
2. Слева нажмите на **⏱ (Выполнения)** (иконка часов)
3. Найдите последнее выполнение функции `onEdit`
4. Кликните на него

**В логах должно быть:**

```
Строка 225: отправка webhook для CRM ID 782
HTTP 200: {"success":true,"queueSize":1,...}
✓ Успех: CRM ID 782 добавлен в очередь. Всего в очереди: 1
✓ Строка 225: webhook успешно отправлен
```

### Тест 3: Проверка обработки в Bitrix24

1. Подождите до следующего часа (:00 минут) или 5 минут (в тестовом режиме)
2. Откройте элемент в Bitrix24 CRM по ID
3. Проверьте, что элемент переместился на стадию **"Оплачено"** (SUCCESS)

**Или проверьте через API:**

```bash
curl -s https://bitrix24-webhook-filter-production.up.railway.app/queue-status
```

Если `queueSize` = 0 → элемент обработан!

## 🐛 Устранение неполадок

### Проблема 1: "Указанных разрешений недостаточно для UrlFetchApp.fetch"

**Причина:** Триггер не авторизован или неправильный OAuth scope.

**Решение:**

1. **Удалите файл appsscript.json** (или удалите секцию `oauthScopes`)
2. **Удалите все триггеры** (⏰ Триггеры → удалить все)
3. **Отзовите разрешения:**
   - Откройте https://myaccount.google.com/permissions
   - Найдите "Отправка платежей в Битрикс" (или название вашего проекта)
   - Нажмите **"Удалить доступ"**
4. **Создайте триггер заново** (см. раздел [Создание триггера](#создание-триггера))
5. **Дайте разрешения** при появлении окна

### Проблема 2: Всплывающие окна появляются при редактировании

**Причина:** Используется старая версия кода с `SpreadsheetApp.getUi().alert()`.

**Решение:**

1. Убедитесь, что вы скопировали **актуальный код** из [google-apps-script.js](google-apps-script.js)
2. В коде НЕ должно быть вызовов `SpreadsheetApp.getUi().alert()`
3. Все уведомления должны быть заменены на `Logger.log()`

### Проблема 3: Скрипт не срабатывает при редактировании

**Причина:** Триггер не создан или не авторизован.

**Решение:**

1. Проверьте страницу триггеров (⏰)
2. Должен быть триггер для функции `onEdit` с событием "При редактировании"
3. Если триггера нет - создайте (см. раздел [Создание триггера](#создание-триггера))
4. Если триггер есть, но не работает - удалите и создайте заново

### Проблема 4: "Ошибка подключения" в логах

**Причина:** Сервер на Railway недоступен или неправильный URL.

**Решение:**

1. Проверьте `SERVER_URL` в конфиге скрипта
2. Должно быть: `https://bitrix24-webhook-filter-production.up.railway.app/sheets-webhook`
3. Проверьте доступность сервера:
   ```bash
   curl https://bitrix24-webhook-filter-production.up.railway.app/health
   ```
4. Если сервер недоступен - проверьте статус на Railway

### Проблема 5: Элемент добавляется в очередь, но не двигается в Bitrix24

**Это НЕ проблема Google Apps Script!**

Скрипт работает правильно - элемент успешно добавлен в очередь на сервере.

**Проблема на стороне сервера Railway.**

См. раздел [Устранение неполадок в README.md](README.md#устранение-неполадок)

## 📊 Как посмотреть логи Apps Script

### Вариант 1: Журнал выполнения (для последнего запуска)

1. В редакторе Apps Script нажмите **Ctrl+Enter**
2. Или: **View → Журнал выполнения**

2. В меню должна появиться новая вкладка **"🔄 Bitrix24 Sync"** (после первого открытия таблицы)
